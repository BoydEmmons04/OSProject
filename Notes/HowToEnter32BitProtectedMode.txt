The bootloader can only be 512 bytes.
This is in real mode

To do more than 512 bytes you cant do bios interrupts and have to switch to
protected mode

Protected mode allows for up to 4GB of memory in 32 bit mode and paging/multitasking
This also lets us use C or C++ if we want

To do this we have to define how segmentation works. 

We have to set variables defining each segment in a data structure called the GDT

GDT
- Global Descriptor Table
- a bit of a mess
- We have to define a descriptor for each segment used in protected mode
- Most use paging
- we are using a flat memory model
- a single contiguous address space
- must contain a code segment descriptor
- must contain a data segment descriptor

Code segment
- must define size and location
- Base - 32 bit start of the segment
- Limit - 20 bit property that should be set to 0xfffff
Descriptor bits
- Present - 1 for if the segment is used
- Privilege - 2 bit value from 00 to 11 with 00 being the highest privilege
- Type - 1 if segment is code or data
- Other properties are stored as flags
- Type flags (4 bits)- 
    1. Code? Is this segment code
    2. Conforming? can this code be executed from lower privilege segments (0 for our use)
    3. Readable? can this segment be read or only executable (1 to read constants)
    4. Accessed? this is managed by CPU so we set to 0
- Other flags - 
    1. Granularity (1 multiplies the limit with 0x1000 so we can use all 4GB of memory)
    2. 32 Bit memory (will this use 32 bit memory 1 for yes)
    3. Not used, 0
    4. Not used, 0

Code Segment Descriptor
- pres, priv, type = 1001
- Type flags = 1010
- other flags = 1100

Data segment descriptor
- (in the data segment, instead of conforming, bit 2 becomes direction, on means expand down)
_ (the readable bit (3) becomes the writable bit, 0 means read only and 1 means read and write)
- pres, priv, type = 1001
- type flags = 0010
- other flags = 1100

To define these in x86 assembly we have to define these in the correct order using:
- db
- dw 
- dd

After in 32 bit protected mode we cannot use bios interrupts
To print to the screen we have to write directly to video memory in 0xb8000
- first byte: character
- second byte: color
